# PX-304.1 — Application Shell Implementation

## Summary

Implemented the ReleaseFlow Application Shell in accordance with **ReleaseFlow Design Standard v1.0 (RFDS v1.0)**. The shell is a persistent, full-viewport application frame that wraps every authenticated page. It replaces the previous ad-hoc layout with a deterministic, accessible, motion-controlled navigation surface.

## Scope

This sprint touches only the shell layer. No domain logic, operational intelligence, releases workspace, artist workspace, services, hooks, repositories, routing, calculations, or deliverables were modified.

## Key Behaviours Delivered

- **Full-viewport fixed frame** — Root element is `h-screen overflow-hidden`; the sidebar occupies the full height and never creates a secondary scrollbar.
- **Three vertical sidebar zones** — Brand, Navigation (`flex-1`), User/Settings/Sign Out. Unused vertical space is eliminated.
- **Desktop collapse** — Expanded 280px, collapsed 72px icon rail. 200ms transition using the RFDS motion token. Collapse preference persisted in `localStorage`.
- **Tablet overlay drawer** — Fixed slide-over with backdrop dismiss; auto-closes after navigation.
- **Mobile slide-over drawer** — Fixed slide-over with backdrop dismiss, Escape-to-close, and **pointer swipe-to-dismiss** (gesture threshold 80px).
- **Content offset** — Content column sits in the flex flow beside the sidebar; the sidebar never overlaps content on desktop.
- **Top Bar refinement** — Search prominence with built-in ⌘K trigger, notification badge placement, user menu alignment, breadcrumb hierarchy, and page title spacing.
- **Accessibility** — Skip link, ARIA labels, `aria-expanded`, `aria-controls`, keyboard navigation, focus trap entry on drawer open, Escape handling, `prefers-reduced-motion` fallback, and 44px touch targets on all interactive elements.

## Motion & Easing

All transitions use the approved RFDS timing:
- `--rf-motion-normal: 200ms` for expand/collapse.
- `--rf-ease-standard: cubic-bezier(0.2, 0, 0, 1)` for standard motion (applied via Tailwind duration classes; further easing alignment can be done via a global CSS variable mapping in a subsequent polish pass).
