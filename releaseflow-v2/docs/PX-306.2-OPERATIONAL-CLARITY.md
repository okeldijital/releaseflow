# PX-306.2 — Operational Clarity Architecture

**Date:** 2026-06-28
**Status:** Complete

---

## Design Principle Applied

> **ReleaseFlow is not a dashboard. It is an Operations Center.**

Every page must answer, in order:
1. What is happening?
2. Does it require my attention?
3. What should I do next?
4. What evidence supports that conclusion?

---

## New Reading Order

```
HERO (Situation)
    ↓
ASSESSMENT (What is the current state?)
    ↓
IMMEDIATE ACTIONS (What should I do now?)
    ↓
METRICS (Supporting evidence — 16px below actions, no separator)
    ↓
ACTIVE RELEASES (Evidence)
    ↓
ATTENTION (If any)
    ↓
ACTIVITY (History)
```

Not:
```
Header
    ↓
Widgets
    ↓
Statistics
    ↓
Table
```

---

## What Changed

### Hero → Situation
- Date remains H1 (40px/medium/tight leading)
- "Operations Center" page title removed — date is the page identity
- Generic status sentence replaced with a **conclusionary briefing** generated from real data:

  | Data State | Briefing |
  |---|---|
  | No releases | "No active releases. Create your first release to begin operational tracking." |
  | All healthy | "All N active releases are healthy. M shipped this month. Operations are stable." |
  | Some at risk | "N releases require attention. M remain healthy. K shipped this month." |
  | All at risk | "N releases require immediate attention. Health is critical across the organisation. Review is recommended." |

- Max 3 lines, plain language, no badges, no pills, no decorative containers
- Max width 640px (65–80 char line)

### Assessment → Current State
- Replaces the floating confidence section
- Two-column editorial grid: Health / Confidence / Current Stage / Deadline
- Typography establishes hierarchy (label uppercase tracking-widest, value 24px/medium/tabular-nums)
- No cards, no borders, no background
- No badges — Health is plain text colored danger if critical
- Max width 640px

### Immediate Actions → What To Do
- **Maximum 3 actions**, priority-ordered
- Each action: one sentence + NOW timestamp right-aligned
- Generated dynamically from data (alerts → blocked → deadlines)
- No icons, no buttons — just text and timestamp
- Large whitespace around the section
- Typography-only (not interactive cards)

### Metrics → Supporting Evidence
- Immediately under actions (16px gap, no separator, no border)
- Inline format: `N active · N blocked · N shipped`
- Smaller typography (text-sm/600 for values, text-[11px] for labels)
- No background, no card — just text on canvas
- Max width 640px

### Evidence → Active Releases
- 40px below metrics
- Full-width table (extends beyond reading column)
- Same columns as before (Release, Health, Stage, Deadline, Owner)
- Remains the primary decision support tool

### Attention → Mission Critical
- Only shown when actionable items exist
- Same structure as before (alerts, blocked, deadlines)
- No visual changes — preserves critical signal

### History → Activity
- Bottom of page, supporting evidence role
- Lower contrast, smaller text
- Same ActivityRow component

### Sidebar Ambience 2.0
- Active state: no background highlight — just darker text
- Hover: only darkens text color, no background
- Section headers: reduced opacity to `text-text-300/70`
- Rounded corners changed from `rounded-lg` to `rounded-md` for tighter feel
