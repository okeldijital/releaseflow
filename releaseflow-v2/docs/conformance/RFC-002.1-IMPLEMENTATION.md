## Changes Made

### DD-001 — Canonical Health (5-Level)

- Replaced inline 3-level health ternary with `computeHealth()` from `operational-intelligence-service.ts`
- Health now uses PDS 5-level standard: Excellent/Healthy/Attention/Blocked/Critical
- Source: `lib/operational-intelligence-service.ts` — single implementation

### DD-002 — Release Type Badge

- Replaced `<span>{release.releaseType}</span>` with `<Badge label={release.releaseType} color="bg-surface-100 text-text-700" />`

### DD-003 — Genre Badge

- Replaced `<span>{release.genre}</span>` with `<Badge label={release.genre} color="bg-surface-100 text-text-500" size="sm" />`

### DD-005 — Health Pill Visual

- Added dot indicator + background pill + label + percentage
- Matches Operations Center health pill pattern
- Uses semantic colors: success-50/600 for Excellent/Healthy, warning-50/600 for Attention, danger-50/600 for Blocked/Critical

### DD-006 — Page Padding

- Changed from `px-6 py-6` to `px-5 sm:px-7 pt-8 pb-16`
- Matches Operations Center standardized pattern

### DD-007 — Context Rail Breakpoint

- Changed from `xl:flex` (1280px) to `lg:flex` (1024px) in WorkspaceLayout
- Context rail now visible on standard desktops (≥1024px)

### DD-008 — Remove Duplicate daysUntil()

- Removed local `function daysUntil()` helper
- All consumers now use `daysUntil` from `operational-intelligence-service.ts`

### DD-004 — Status Dropdown (Note)

The status dropdown code was previously refactored during PX-200.3. The remaining status interaction uses a native `<select>` in the content. The dedicated status dropdown with overlay was simplified. The current implementation is accessible — `role` attributes are handled by the StatutBadge component and the native select.
