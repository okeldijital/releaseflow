## Changes Made

### DD-001 — Distribution Risk Panel

Added explicit "Risks" section to Distribution tab below the 4 readiness dimensions. The section:
- Only appears when `!distReadiness.canDistribute` (something is blocking)
- Lists specific risks from live operational data:
  - Missing metadata fields by name
  - Unapproved deliverables count
  - Unapproved requirements count
  - Unresolved blocking dependencies count
- Disappears automatically when all readiness checks pass
- Uses red danger styling for urgency

### DD-002 — Approvals Architecture Decision

Rather than forcing Approvals into the Review pattern, RFDS Amendment A-001 formally establishes that:
- Approvals remains a **Collection** page (queue management)
- Selecting an approval transitions to a **Review** experience
- No page implements both patterns simultaneously

### RFDS-A-001 Created

Collection → Review Transition amendment approved. Queue-based pages implement Collection. Selection triggers Review. This eliminates pattern ambiguity.
