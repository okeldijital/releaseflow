## What Changed

### Real Release Data
The Release Health Table now uses live data from `fetchOperationsData()` instead of hardcoded rows. The service fetches releases from the repository, computes health percentages per release, and returns structured `ReleaseSummaryRow[]`.

### Org Pulse
5 stat cards added per v3 spec: active releases, blocked stages, overdue deadlines, over budget, shipped this month. Each card uses semantic coloring (red for >0 blocked/overdue, green for shipped).

### Section Ordering (v3 spec)
```
Page Header (title + date + CTA)
  ↓
Operational Summary (domain-ui)
  ↓
Active Releases (Table)
  ↓
Attention Panel (Alerts + Blocked + Deadlines) — hidden when empty
  ↓
Org Pulse (5 stat cards)
  ↓
Recent Activity (chronological feed)
  ↓
Quick Actions (role-aware)
  ↓
Footer (refresh + updated time)
```
