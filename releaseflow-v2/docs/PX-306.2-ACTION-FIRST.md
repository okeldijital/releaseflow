# PX-306.2 — Action-First Layout

**Date:** 2026-06-28

---

## Principle

Actions are not widgets. They are instructions.

Each action should read like a sentence the system is telling you to follow.

---

## Format

```
Immediate Actions

Review advertising budget exceeded and resolve    NOW
Complete readiness checklist for blocked work       NOW
Address 3 upcoming deadlines before shipment        NOW
```

## Rules

- **Maximum 3 actions** — beyond that, the user cannot act on all of them
- Priority-ordered: alerts → blocked → deadlines
- One action per row
- Each action: one sentence in present tense
- NOW timestamp right-aligned in monospace uppercase
- No icons, no buttons, no backgrounds
- 16px between actions
- Section header uppercase tracking-widest
- Generated dynamically from data

## Generation Logic

```typescript
function generateActions({ alerts, blockedItems, deadlines, releases }) {
  const actions = [];
  if (alerts.length > 0) {
    actions.push({ text: `Review ${alerts[0].message} and resolve` });
  }
  if (blockedItems.length > 0) {
    actions.push({ text: 'Complete readiness checklist for blocked work' });
  }
  if (deadlines.length > 0) {
    actions.push({ text: `Address ${deadlines.length} upcoming deadline(s) before shipment` });
  }
  if (actions.length === 0 && releases.length > 0) {
    actions.push({ text: 'Review release status and prepare for distribution' });
  }
  return actions.slice(0, 3);
}
```

## Result

The user knows exactly what to do. No buttons to evaluate. No decision trees to navigate. Just the next action.
