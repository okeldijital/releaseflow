# PX-306.2 — Operational Language Guidelines

**Date:** 2026-06-28

---

## Principle

The interface should summarize, not narrate database values.

Every sentence should be a conclusion, not a fact.

---

## Avoid

```
Release is in critical state. 0 of 1 stages complete.
Currently in Operations. 0 of 1 readiness checks passed.
```

This is a database dump. The user has to interpret it.

## Prefer

```
This release is at risk. No readiness checks have passed
and the Operations stage has not been completed.
Immediate review is recommended.
```

This is a conclusion. The system has done the interpretation.

---

## Rules

1. **State conclusions, not values.** "Health is critical" not "Health: 15%"
2. **Explain "why" once.** The user needs context, not data.
3. **Use "no", "not", "unaddressed"** to indicate absence. The user needs to know something IS missing.
4. **Suggest action.** Every observation implies what to do about it.
5. **Maximum 3 sentences per statement.** Anything more is a paragraph.
6. **65–80 characters per line.** Reading width matters.

---

## Patterns

| Pattern | Example |
|---------|---------|
| State status | "This release is at risk." |
| Explain absence | "No readiness checks have passed." |
| Identify stage | "The Operations stage has not been completed." |
| Suggest action | "Immediate review is recommended." |
| Acknowledge health | "All N active releases are healthy." |
| Quantify when needed | "N shipped this month." |

---

## Examples Per Section

### Hero
- "1 release requires immediate attention."
- "Health is critical across the organisation."
- "Review is recommended."

### Immediate Actions
- "Review advertising budget exceeded and resolve."
- "Complete readiness checklist for blocked work."
- "Address 3 upcoming deadlines before shipment."

### Assessment
- "Health: Critical" (not "Health: 15%")
- "Current Stage: Operations"
- "Deadline: 7 days"

### Metrics
- "N active"
- "N blocked"
- "N shipped this month"
