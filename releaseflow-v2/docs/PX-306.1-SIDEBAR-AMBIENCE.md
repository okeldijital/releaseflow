# PX-306.1 — Sidebar Ambience

**Date:** 2026-06-28

---

## Principle

The sidebar should feel as though it sits directly on the application canvas — not as a competing panel, but as a utility that recedes into the background until called upon.

---

## Before

```
┌───────────────┬────────────────────────────────────┐
│ bg-surface-50 │  Content area                        │
│ border-r      │                                     │
│               │                                     │
│ ReleaseFlow   │                                     │
│ ─────────     │                                     │
│ Operations    │                                     │
│   Home        │                                     │
│   Releases    │                                     │
│ ─────────     │                                     │
│ Resources     │                                     │
│   Artists     │                                     │
│   Assets      │                                     │
│ ─────────     │                                     │
│ border-t      │                                     │
│ User          │                                     │
│ Sign out      │                                     │
└───────────────┴────────────────────────────────────┘
```

**Visual weight**: The sidebar's background (`surface-50`) and border (`surface-200`) made it visually compete with content. The footer separator (`border-t`) added unnecessary visual division within the sidebar itself.

---

## After

```
┌───────────────┬────────────────────────────────────┐
│ transparent   │  Content area                        │
│ no border     │                                     │
│               │                                     │
│ ReleaseFlow   │                                     │
│               │                                     │
│ Home          │                                     │
│ Releases      │                                     │
│ Artists       │                                     │
│ Assets        │                                     │
│ Admin         │                                     │
│               │                                     │
│ User          │                                     │
│ Sign out      │                                     │
└───────────────┴────────────────────────────────────┘
```

**Visual weight**: Transparent, borderless. The sidebar is discernible only by its position and content.

---

## Changes

| Property | Before | After |
|----------|--------|-------|
| Background | `bg-surface-50` | Removed (transparent) |
| Border right | `border-r border-surface-200` | Removed |
| Footer separator | `border-t border-surface-200` | Removed |
| Active nav item | `bg-primary-50/60` | `bg-primary-50/40` |
| Hover nav item | `bg-surface-100/30` | `bg-surface-100/20` |
| Nav item height | `min-h-[44px]` | `min-h-[40px]` |
| Nav item vertical padding | `py-2` | `py-1.5` |

---

## Behavior

| State | Expanded (280px) | Collapsed (72px) |
|-------|-----------------|-------------------|
| Default | Icons + labels + section headers | Icons only |
| Hover | Subtle background on active row | Tooltip appears |
| Active | Softer highlight, text colored | Same icon state |
| Collapse toggle | Chevron at right edge | Chevron reversed |

No shadows. No border. No elevation. The sidebar is a utility, not a destination.
