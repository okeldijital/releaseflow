# PX-306.2 — Hero Briefing

**Date:** 2026-06-28

---

## Before

```
Date string (text-2.5rem)
Status line: "0 healthy · 1 at risk · 0 shipped this month"
```

## After

```
Date string (text-2.5rem)
───
"1 release requires immediate attention. Health is critical across
the organisation. Review is recommended."
```

---

## Requirements

- Date remains H1 (text-2.5rem, medium, tracking-tight)
- Briefing replaces generic summary sentence
- Maximum 3 lines
- Plain language — no bullet syntax
- No badges, no pills, no decorative containers
- No icons

## Rules

- Generate briefing from real data
- Conclusions not raw facts
- Acknowledge healthy state, don't just list counts
- Escalate to "review is recommended" when health is critical
- Max width 640px for readability

## Generation Logic

```typescript
function generateBriefing({ healthyCount, atRiskCount, shipped, blockedItems, deadlines }) {
  if (healthyCount === 0 && atRiskCount > 0) {
    return `${atRiskCount} release(s) require immediate attention. Health is critical across the organisation. Review is recommended.`;
  }
  if (atRiskCount > 0) {
    return `${atRiskCount} release(s) require attention. ${healthyCount} remain healthy. ${shipped} shipped this month.`;
  }
  if (healthyCount > 0) {
    return `All ${healthyCount} active release(s) are healthy. ${shipped} shipped this month. Operations are stable.`;
  }
  return 'No active releases detected. Operations are idle.';
}
```

## Result

The user no longer needs to count and compare. The system tells them what to do.
